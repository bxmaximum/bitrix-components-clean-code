# Lesson 01: Legacy-компонент (исходная версия)

Это типичный компонент, написанный в стиле 2015 года. Он работает, но содержит множество архитектурных проблем.

## Антипаттерны в этом компоненте

### component.php

| # | Проблема | Строка | Последствия |
|---|----------|--------|-------------|
| 1 | Глобальные переменные | 18 | Неявные зависимости, сложно тестировать |
| 2 | Нет валидации параметров | 21-23 | Возможны ошибки при неверных данных |
| 3 | Старый API `CModule::IncludeModule` | 26 | Нет D7, нет автозагрузки |
| 4 | Ручное кэширование без `additionalCacheId` | 30-33 | Cache Poisoning! |
| 5 | `CIBlockElement::GetList` вместо ORM | 45 | Устаревший API, нет типизации |
| 6 | N+1 запросы (автор в цикле) | 67-72 | На 50 новостей = 51 запрос к БД |
| 7 | Смешение логики и форматирования | 57-65 | Нарушение SRP |
| 8 | SEO захардкожен | 87-88 | Нельзя переиспользовать |

### template.php

| # | Проблема | Строка | Последствия |
|---|----------|--------|-------------|
| 1 | Запросы к БД в шаблоне | 42-46, 73-77 | N+1, нарушение MVC |
| 2 | Бизнес-логика (`$USER->IsAdmin()`) | 64 | Шаблон не должен принимать решения |
| 3 | Inline-стили | везде | Невозможно переопределить |
| 4 | Нет экранирования | 32, 36 | XSS-уязвимость |

## Как проверить проблемы

1. Откройте Debug-панель Битрикса
2. Посмотрите количество SQL-запросов
3. При 10 новостях вы увидите ~25+ запросов вместо 1-2
